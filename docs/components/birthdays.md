# Дни рождения и напоминания

Файл `src/birthday/service.js` отвечает за регистрацию дат, календарь чата, тексты уведомлений и фоновую проверку наступивших событий.

## Команды

| Команда | Действие |
| --- | --- |
| `/birthday ДД.ММ.ГГГГ` | Создать или обновить день рождения автора команды в текущем групповом чате |
| `/birthday ДД.ММ` | То же без сохранения года рождения |
| `/bday ...` | Короткий псевдоним `/birthday` |
| `/birthdays`, `/birthday_list` | Показать календарь текущего чата |
| `/birthday_remove`, `/birthday_delete` | Удалить дату автора из текущего чата |

Разделители `.`, `/` и `-` равнозначны. Год, если указан, должен находиться между 1900 и текущим годом. Повторная регистрация обновляет дату и актуальные Telegram-имя, username и название чата.

Команды работают только в `group` и `supergroup`: одна и та же учётная запись может хранить разные даты в разных чатах.

## Ежедневная проверка

`src/server/index.js` вызывает `startScheduler()` один раз при старте постоянного Node.js-процесса. Сервис сразу выполняет проверку, затем повторяет её через `BIRTHDAY_CHECK_INTERVAL_MS`.

Проверка ничего не отправляет до `BIRTHDAY_CHECK_HOUR` в часовом поясе `BIRTHDAY_TIME_ZONE`. После контрольного часа:

1. для сегодняшних именинников отправляется праздничное HTML-сообщение в групповой чат;
2. для завтрашних именинников другим зарегистрированным участникам этого чата отправляются личные напоминания;
3. уникальная запись в `birthday_notifications` предотвращает повторную отправку того же события при следующем tick или параллельном процессе.

Для 29 февраля в невисокосном году события выполняются 28 февраля. Старые маркеры доставки удаляются через 400 дней.

Vercel Function создаёт Birthday Service для обработки команд, но не запускает interval: serverless-процесс не гарантирует постоянную работу. Автоматические поздравления рассчитаны на `npm start`, в частности на Railway service.

## Ограничение Telegram для личных сообщений

Telegram не разрешает боту первым начинать личный диалог. Каждый участник, желающий получать напоминания, должен открыть бота в ЛС и нажать Start. Получателями являются другие пользователи, которые зарегистрировали свой день рождения в том же чате; имениннику напоминание о нём самом не отправляется.

Постоянные ошибки Telegram (`Forbidden`, блокировка бота, удалённый пользователь или отсутствующий private chat) фиксируются в логах без повторов. При временной ошибке marker удаляется, поэтому следующий tick попробует отправить уведомление ещё раз.

## Конфигурация

| Переменная | По умолчанию | Назначение |
| --- | --- | --- |
| `BIRTHDAY_SCHEDULER_ENABLED` | `true` | Значение `false` отключает автоматические отправки, но оставляет команды |
| `BIRTHDAY_TIME_ZONE` | `Asia/Almaty` | IANA timezone для календарной даты |
| `BIRTHDAY_CHECK_HOUR` | `9` | Первый локальный час отправки, 0…23 |
| `BIRTHDAY_CHECK_INTERVAL_MS` | `900000` | Интервал повторной проверки, минимум 60 секунд |

Для команд и планировщика обязателен `DATABASE_URL`. Таблицы создаются автоматически при старте.
