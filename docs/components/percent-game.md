# Игра `/percent`

Percent Game отвечает случайной фразой о том, на сколько процентов игрок соответствует выбранному параметру.

## Команда и выбор игрока

```text
/percent gay
```

Без reply игра оценивает автора команды. Если команда отправлена в ответ на сообщение, оценивается автор этого сообщения. Для пользователя с `username` подставляется `@username`; иначе используются имя и фамилия.

Команда принимает ровно один параметр. Пустая команда показывает синтаксис и список доступных параметров, неизвестное значение — тот же список. Суффикс имени бота поддерживается общим parser, например `/percent@my_bot gay`.

## Конфигурация

Параметры, aliases и варианты ответов находятся в `config/percent-game.json`. При запуске обе точки входа читают и валидируют файл. Изменения требуют перезапуска процесса или нового serverless deployment.

Верхнеуровневые поля:

| Поле | Назначение |
| --- | --- |
| `command` | ASCII-имя команды без `/` |
| `ttlSeconds` | Срок фиксации результата; по умолчанию `86400` |
| `messages` | Тексты usage, неизвестного параметра и ошибок |
| `parameters` | Объект доступных параметров |

Каждый параметр содержит `aliases` и непустой массив `phrases`. Имя параметра и каждый alias должны быть одним токеном. Один alias нельзя назначить нескольким параметрам.

В шаблонах фраз поддерживаются placeholders:

- `{user}` — `@username` или отображаемое имя игрока;
- `{percent}` — целое число от 0 до 100;
- `{parameter}` — каноническое имя параметра.

В служебных сообщениях доступны `{command}` и `{parameters}`. Чтобы добавить новый тип, достаточно добавить объект в `parameters`; код менять не нужно.

## Суточное хранение

Percent Game использует Upstash Redis. Ключ имеет форму:

```text
percent-game:v1:<userId>:<parameter>
```

В значении сохраняются процент и выбранный шаблон, а TTL берётся из `ttlSeconds`. Chat ID намеренно не входит в ключ: один Telegram-пользователь получает один результат для одного параметра во всех чатах до истечения TTL. Первичная запись использует `NX`, поэтому параллельные вызовы перечитывают победившее значение.

Без Redis команда отвечает, что игра временно недоступна: несохранённый случайный результат не выдаётся, чтобы не нарушать суточный контракт.
