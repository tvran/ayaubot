# Аналитика, токенизация и игры

Подсистема состоит из чистого токенизатора `src/analytics/tokenize.js` и сценарного сервиса `src/analytics/service.js`.

## Токенизатор

### `normalizeWord(word)`

Приводит строку к нижнему регистру, заменяет `ё` на `е` и удаляет один или несколько апострофов/обратных кавычек только по краям.

### `tokenizeText(text = '')`

Возвращает массив слов. Алгоритм:

1. пустой текст и текст, начинающийся с `/`, дают пустой массив;
2. HTTP/HTTPS URL удаляются;
3. Unicode-регулярное выражение извлекает последовательности букв и цифр; внутри допускаются апострофы и дефис;
4. применяется нормализация;
5. удаляются слова короче трёх символов и встроенные русские, казахские и английские stopwords.

Стемминга, лемматизации и определения языка нет. Поэтому словоформы считаются разными словами, а дефис сохраняется.

### `wordCounts(words)`

Сворачивает массив в `Map<word, count>`, используемый для одного пакетного upsert в PostgreSQL.

## Analytics Service

```js
createAnalyticsService({ db })
```

Сервис допускает отсутствие DB adapter. В этом режиме методы команд возвращают сообщение о ненастроенной аналитике, а `ingestMessage` и `checkCodewordGuess` ничего не делают.

Текст сообщения извлекается как `message.text || message.caption || ''`. Имя пользователя строится из `first_name` и `last_name`, затем используется username, затем ID.

## Сбор статистики

### `ingestMessage(message)`

Требует `message.from` и `message.chat`. Сначала всегда обновляет профиль пользователя. Если после токенизации есть слова, увеличивает дневные счётчики. Дата берётся из Unix timestamp Telegram `message.date`, а при его отсутствии — из текущего времени процесса.

Edited message повторно увеличит счётчики; компенсации предыдущей версии сообщения нет.

### `topWordsText(chatId, days = 14)`

Запрашивает пять слов и лидера-пользователя для каждого слова. Возвращает готовый plain-text ответ. Пустой результат сопровождается сообщением о недостатке данных.

## Игра с кодовым словом

`ensureActiveCodeword(chatId)` — внутренний общий механизм:

- возвращает активную неистёкшую игру;
- помечает истёкшую игру как `expired`;
- при отсутствии игры случайно выбирает одно слово из топ-15 за 14 дней;
- создаёт игру сроком на три дня.

Из этого следует, что `/codeword`, `/codeword_hint` и `/codeword_start` способны автоматически создать игру. Команда start не является единственным способом запуска.

Методы:

- `startCodewordText` создаёт/находит игру и сообщает о старте;
- `codewordStatusText` показывает округлённое вверх число оставшихся часов;
- `codewordHintText` показывает `active.word.length` и лидера по слову;
- `stopCodewordText` помечает текущую игру `expired`;
- `checkCodewordGuess` проверяет обычное сообщение, завершает игру и возвращает победный текст.

Угадывание проверяется по токенам на точное совпадение нормализованного слова. Подстроки не засчитываются. Команды не проверяются. JavaScript `length` считает UTF-16 code units, а не пользовательские графемы; для обычной кириллицы результат соответствует числу букв.

## Ежедневный выбор

### `pidorOfDayMessages(chatId, botId)`

Вызывает `db.dailyPick(chatId, 'pidor', [botId])`. Сохранённый на сегодня пользователь возвращается повторно; иначе выбирается случайный пользователь из таблицы `users`, кроме бота. Метод возвращает массив из семи сообщений для последовательной отправки.

В выбор попадают все пользователи, которые ранее были записаны через `ingestMessage`; проверки давности активности нет.

### `resetPidorText(chatId)`

Удаляет сегодняшний результат вида `pidor`, после чего следующий вызов выберет заново.

### `pidorHistoryText(chatId)`

Формирует список последних 15 результатов, новые даты идут первыми.

## Контракт DB adapter

Сервис ожидает методы:

```text
upsertUser              incrementWordCounts
topWords                topUsersForWords
activeCodeword          createCodeword
expireCodeword          guessCodeword
dailyPick               resetDailyPick
dailyPickHistory
```

Это позволяет тестировать сервис с in-memory adapter, хотя готового adapter и тестов в репозитории нет.
